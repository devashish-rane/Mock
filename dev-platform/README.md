# dev-platform

Simple shared local infrastructure workspace for LocalStack Free and AWS CDK.

## What this is for

- Run one LocalStack instance for multiple app repos
- Define shared local AWS resources once with CDK
- Keep app repos like `QCAFE/` separate from infra lifecycle

## Layout

```text
dev-platform/
  Makefile
  .env.example
  infra-cdk/
  outputs/
    local/
```

## Prerequisites

- Docker
- LocalStack CLI
- Node.js
- npm
- Global CDK CLIs:

```bash
npm install -g localstack aws-cdk aws-cdk-local
```

or install the LocalStack CLI separately from the official release instructions.

```bash
npm install -g aws-cdk aws-cdk-local
```

`cdklocal` is the LocalStack wrapper around the AWS CDK CLI.

## Quick start

```bash
cd dev-platform
cp .env.example .env
make up
make status
cd infra-cdk && npm install
make bootstrap
make deploy
```

The startup logic lives in `scripts/start-localstack.sh` so the `Makefile` stays thin.
The script relies on LocalStack's default ports, reads overrides from `.env`, and always starts LocalStack in free mode.

## Resources created

- 1 shared S3 bucket
- 1 shared SQS queue
- 1 shared DynamoDB table

## Notes

- This setup is intentionally basic and stays inside LocalStack Free-friendly services.
- Avoid CDK constructs that depend on custom resources or Lambda-backed asset helpers for local-only stacks.
- If you later want per-app stacks, add new files under `infra-cdk/lib/` and wire them in `bin/dev-platform.ts`.
- Resource names are generated by CloudFormation/CDK. App repos should consume stack outputs instead of hard-coding names.
- The current stack only needs `cloudformation,s3,sqs,dynamodb,sts,iam,ssm`. `ecr` is not needed unless you later add container-image asset flows.

## Useful commands

```bash
make up
make status
make logs
make bootstrap
make deploy
make destroy
make show-outputs
make down
```

## Troubleshooting

- If CDK fails with `ECONNREFUSED ::1:4566`, use `127.0.0.1` instead of `localhost` for the LocalStack endpoint to avoid IPv6 resolution issues.
- If you see `current credentials could not be used to assume arn:aws:iam::000000000000:role/cdk-...`, that is usually harmless in LocalStack when the account is already `000000000000`.
- If deploy fails, first confirm LocalStack is running:

```bash
localstack status
localstack logs -f
```

- If a previous failed deploy left the stack in `ROLLBACK_COMPLETE`, delete the failed stack before deploying again:

```bash
aws --endpoint-url=http://127.0.0.1:4566 cloudformation delete-stack --stack-name SharedLocalStack
```

- The official LocalStack CLI starts the runtime in Docker by default. Current LocalStack docs mark direct host mode as deprecated, so this repo uses the CLI instead of Docker Compose, not a true non-container runtime.
- This repo does not support token-backed Pro/UI setup. It is intentionally free-mode only.
